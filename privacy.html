provisions related to api for google user data

Who is Requesting Google User Data? When you log into our extension for the first time, you will be asked to grant permission for the extension to access and use the user data provided through the Google API. The identity of the extension will be shown as ZEFR.

What Data is the Application Requesting? There are two types of Google user data accessed through the API. The first type of data is campaign data, which consists of your targeting details, your ad details including ID’s of associated videos and ads, and the names and ID’s associated with the different levels (parent and child) of ad campaigns. The bulk of the data passed through the Google API consists of this campaign data.

The second type of data accessed via the Google API is user personal data. User personal data consists of first name, last name, email address, browser type, user ID, IP address, and similar data.

All user data (campaign data and user personal data) is accessed using a third party, Auth0, so we do not ever have access to that data, except to provide the services we’ve described.

Why do We Request Google User Data? We request the Google campaign data to provide the services you request using the extension. We use the user personal data to verify that the user is actually an authorized user.
